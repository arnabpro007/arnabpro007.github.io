Linear Methods for Regression¶

ESL Chapter-3¶

Introduction:

A linear regression model assumes that the regression function E[Y|X] is linear in the inputs. Linear models were largely developed in the pre-computer era of statistics, but even in today's computer era they are still good reasons to study an use them. They are simple and often provide an adequate and interpretable description of how the inputs affect the output.

For prediction purposes they can sometimes outperform fancier non linear models, especially in situations with small number of training cases, low signal to noise ration or sparse data. Finally, linear metods can be applied to transformations of the inputs and this considerably expands their scope.
We will start with the simplest linear method that is multiple linear regression.

def linearity_test(model, y):
    '''
    Function for visually inspecting the assumption of linearity in a linear regression model.
    It plots observed vs. predicted values and residuals vs. predicted values.
    
    Args:
    * model - fitted OLS model from statsmodels
    * y - observed values
    '''
    fitted_vals = model.predict()
    resids = model.resid

    fig, ax = plt.subplots(1,2)
    
    sns.regplot(x=fitted_vals, y=y, lowess=True, ax=ax[0], line_kws={'color': 'red'})
    ax[0].set_title('Observed vs. Predicted Values', fontsize=16)
    ax[0].set(xlabel='Predicted', ylabel='Observed')

    sns.regplot(x=fitted_vals, y=resids, lowess=True, ax=ax[1], line_kws={'color': 'red'})
    ax[1].set_title('Residuals vs. Predicted Values', fontsize=16)
    ax[1].set(xlabel='Predicted', ylabel='Residuals')
    
linearity_test(model, Y)

So as the datas are more or less symetrically placed around the diagonal line we can claim that the linearity assumption was a valid assumption

Subset Selection Methods:

There can be thousands of reasons to be disgusted with least square estimates but there are two main reason which causes those!

  • Prediction Accuracy: Let's be honest here, I haven't seen a single case were OLS estimates ends up getting the best prediction accuracy (apart from some tailor made models maybe).The reason for this is least square estimates have low bias and large variance. Prediction accuracy can sometimes be improved by shrinking or setting some co-efficients to be 0
  • Interpretation: With a large number of predictors, we often would like to determine a smaller subset that exhibit the strongest effects. In order to get the big picture, we are willing to sacrifice some of the small details
There are quite a few methods to accomplish this goal and we will cover most of them here.

Best Subset Selection:

Best subset regression finds for each k {0, 1, 2, . . . , p} the subset of size k that gives smallest residual sum of squares. An efficient algorithm the leaps and bounds procedure (Furnival and Wilson, 1974)makes this feasible for p as large as 30 or 40. Now note that the best subset curve is necessarily decreasing, so cannot be used to select the subset size k. The question of how to choose k involves the trade-off between bias and variance. When all said and done this is not a method we will use in general , so if you want to use it best is to write your own code. I will upload a general code for this one(atleast for the case when p is less than 40), keep looking at my website it will be posted soon.

Forward and Backward Stepwise Selection:

From one of the least used method to one of the heavily used method, Forward and backward stepwise selection is one of the most applied method. Rather than search through all possible subsets, we can seek a good path through them. Forward stepwise starts with the intercept and then sequentially adds into the model predictor that improves the fit. Backward stepwise starts with the full model and sequentially deletes the predictor that has the least impact on the fit. The candidate for dropping is the variable with the smallest z-score.
Backward selection can only be used when N>p, while forward selection can always be used. Finally,note that often variables come in groups (dummy variables). Smart stepwise procedure (such as step in R) will add or drop whole groups at a time, taking proper account of their degrees of freedom.

#Forward Selection using manual function
import statsmodels.api as sm
def forward_selection(data, target, significance_level=0.05):
    initial_features = data.columns.tolist()
    best_features = []
    while (len(initial_features)>0):
        remaining_features = list(set(initial_features)-set(best_features))
        new_pval = pd.Series(index=remaining_features)
        for new_column in remaining_features:
            model = sm.OLS(target, sm.add_constant(data[best_features+[new_column]])).fit()
            new_pval[new_column] = model.pvalues[new_column]
        min_p_value = new_pval.min()
        if(min_p_value<significance_level):
            best_features.append(new_pval.idxmin())
        else:
            break
    return best_features

#Backward Selection Using manual function
def backward_elimination(data, target,significance_level = 0.05):
    features = data.columns.tolist()
    while(len(features)>0):
        features_with_constant = sm.add_constant(data[features])
        p_values = sm.OLS(target, features_with_constant).fit().pvalues[1:]
        max_p_value = p_values.max()
        if(max_p_value >= significance_level):
            excluded_feature = p_values.idxmax()
            features.remove(excluded_feature)
        else:
            break 
    return features

Forward Stagewise Regression

Forward stagewise regression is even more constrained than forward stepwise regression, with an intercept equals mean(y) , and centered predictors with co-efficients initially all 0. At each step the algorithm identifies the variable most correlated with the current residual. It then computes the simple linear regression co-efficient for that variable. This continue till none of the variables have correlation with residuals. This method pays dividend in high dimensional problems. This method also needs custom made implementation , I will provide one soon so keep looking at my home page

Shrinkage Methods:

By retaining a subset of the predictors and discarding the rest,subset selection produces a model that is interpretable and has possibly lower prediction error than the full model. However, because it is a discrete process - variables are either retained or discarded - it often exhibit high variance. Shrinkage methods are more continuous , and don't suffer as much from high variability.

Ridge Regression:

Ridge regression shrinks the regression coefficients by imposing a penalty on their size.The ridge coefficients minimize a penalized residual sum of squares. It takes into account L2 norm. For more details on Ridge Regression check here: https://en.wikipedia.org/wiki/Tikhonov_regularization
