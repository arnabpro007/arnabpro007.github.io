This is the HTML description file of Control Flow statements in Java. For code check the github repo.

Switch File:

  • Switch is used when you are testing same variable. break is used inside switch statements to prevent the code from going inside same type of loops. Using break properly is very important.
  • The last break is optional but best practice is to use it.
  • Switch statement can be used for 4 primitive types byte,short,int, char
  • But you have JDK 7 or later use can use switch on Strings as well.
  • We can chain the case statements inside switch that makes the code a lot simpler.

For Loop File

  • When we want to run same chunk of code with some particular range of values we will use for loop.
  • Idea is for statement will repeatedly loop over a code chunk until some condition is satisfied.
  • We have to pass start, end and increment values.
  • for(init ; condition ; increment){
    //Code
    }
  • we can increment by any amount and in any direction we want.
  • Be very careful while declaring for loop conditions, it is very common to run into infinite loop which might cause your IDE to crash.

While Loop File

  • We use while when we are not sure how many exact steps our end condition needs.
  • While loop also needs an ending condition. A point of caution it is even easier to run into infinte loops in while loops. So beware of that.
  • We can also break out of the while loop using a break statement.
  • Do-While loop is guaranteed to execute atleast once. Again use the incremet and termination condition very carefully.
  • Continue is a very powerful keyword in Java. If you put continue anywhere inside a while loop then the loop will skip that pass and directly move to the next step. Note that this is almost as strong as break keyword.
  • Difference between while and do-while is in while we test the condition at top and in do while at the bottom.
  • while(condition){
    //Statements
    }
  • do{
    // Statements
    } while(condition);

Parsing Values File

  • Parsing methods helps us to convert string into various primitive type.
  • Integer.parseInt is used to parse a string to integer. If the parsing method fails we will get an error. So be very careful while using it.
  • Parsing is very useful when we take input as string and then change it to int and perform different calculaions.
  • To change string to double pass Double.parseDouble, again be very careful on what argument you pass
  • We can also use parse for float, long , short etc

Reading User Input:

  • Scanner is a class that allows to read input directly from console.
  • A new instance of scanner: Scanner scanner = new Scanner(System.in);
  • scanner.nextLine() reads the next line input.
  • Make sure to close scanner once you have finished using it. It releases the extra memory.
  • To get an int you have to use scanner.nextInt() , similarly different commands for boolean , byte, short, char etc
  • If you are reading int and then a string(nextLine) make sure to put one buffer nextLine otherwise you will get an error ( this error arises from the fact that enter is treated as a line itself).
  • We can use hasNextInt to check if the inout is at all an int
