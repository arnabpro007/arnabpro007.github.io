This is the HTML description file of Object Oriented Programming Module 2 in Java. For code check the github repo.

  • Composition is the idea that different pieces will add up to create one giant chunk.
  • In JAVA programming we create different small objects and then we create one object that contains all of these classes.
  • Composition is fundamentally different from inheretence as in you can only extend to one class there but in composition you can include as many classes as you wish.
  • A general rule of thumb is try to use composition if it doesn't work only then move to inheritence.
  • Encapsulation is the process to prevent outside classes to access the code. It is very important to use it otherwise our code might break down.
  • The issue is if you change anything in the class it affects the main. It won't be an issue for a small code but for a big library it really becomes painful.
  • Polymorphism allows actions to act differently on the object that the action is performed on.
